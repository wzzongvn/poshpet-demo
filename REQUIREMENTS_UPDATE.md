# PoshPet 추가 요구사항 명세 (2026-05-29 사장님 검수)

> 이 문서의 항목들은 현재 `index.html` 프로토타입뿐 아니라 **향후 실제 개발(Next.js 등) 시에도 동일하게 반영**되어야 하는 확정 요건이다.
> 핸드오프 문서(HANDOFF_PROMPT.md)의 "데모 후 의견 반영" 섹션에 연결된다.

---

## R1. 고양이 광견병 백신 항목 제거
- **요건**: 반려동물 종이 **고양이**이면 백신 입력/표시에서 **광견병(Rabies) 항목을 노출하지 않는다.** 종합백신(FVRCP)만 묻는다.
- **이유**: 매장 운영 기준상 고양이는 광견병 미대상.
- **개발 시 주의**: 백신 상태 판정 로직(`vaccinesSummary`)도 고양이는 rabies를 판정에서 제외해야 한다. (개=rabies+core 둘 다, 고양이=core만)
- **영향 범위**: 펫 등록/수정 시트, 펫 상세 카드, 예약 확인 백신 안내, 관리자 백신 알림.

## R2. 로그인 / 회원가입 UI 분리
- **요건**: "로그인"과 "회원가입(간편 가입)"의 화면이 지금은 동일하다. **두 화면의 UI/문구를 구분**한다.
  - 로그인: 기존 회원이 다시 들어오는 맥락 (환영 문구·"로그인" 표현)
  - 가입: 신규 회원 (혜택 안내·"가입하고 시작" 표현)
- **개발 시**: 진입점(`openAuth`)에서 mode('login'|'signup')를 구분해 타이틀/리드/CTA 문구를 다르게.

## R3. 로그인 수단 기억 (default 표시)
- **요건**: 한번 어떤 방법(Kakao/Zalo/Google/전화)으로 로그인하면, 다음부터 **그 수단을 기본(default)으로 크게** 보여주고, 나머지 수단은 "다른 방법으로 로그인" 형태로 접어둔다.
- **개발 시**: 마지막 사용 provider를 저장(localStorage 또는 서버 프로필). 데모에서는 상태 변수로.

## R4. 픽업 옵션 기본값 = "네, 픽업 받을게요"
- **요건**: 예약 흐름의 픽업 단계에서 기본 선택을 **픽업 받기(yes)**로 한다. (하노이 럭셔리 펫케어 특성상 픽업 수요가 높음)

## R5. 예약 확정 후 "예약 수정" 바로가기 버튼
- **요건**: 예약 완료(vDone) 화면 하단에 **방금 한 예약을 바로 수정**할 수 있는 버튼 추가.
- **개발 시**: 기존 `changeBooking(id)` 로직 재사용.

## R6. 예약 버튼 진입 분기 (기존 예약 있으면 최근 예약 먼저)
- **요건**: 하단 "예약(+)" 버튼을 눌렀을 때
  - 예약이 **하나도 없으면**: 지금처럼 서비스 목록 → 신규 예약.
  - 예약이 **한 건이라도 있으면**: 화면 상단에 **가장 최근 예약 1건을 카드로** 보여주고, 그 카드를 누르면 **예약 현황(마이펫/예약 목록)**으로 이동해 전체 예약을 보여준다. 그 아래에 신규 예약 진입도 유지.

## R7. 픽업 주소 입력 시 "예약 확인" 버튼 활성화 색 반영
- **요건**: 픽업="네"에서 주소별칭+상세주소를 입력하면 **예약 확인 버튼이 활성(색 변경)**되어야 한다. (지금은 진행은 되나 비활성 색으로 보여 혼란)
- **개발 시**: 폼 유효성(주소 입력 완료) → 버튼 enabled 상태 동기화.

## R8. 픽업 단계 레이아웃 — 하단 입력 가림 해결
- **요건**: 픽업에서 기존 주소를 선택/추가하면 항목이 늘어나 **연락받을 번호 등 하단 요소가 고정 버튼/탭바에 가려져 선택 불가**. (첨부 이미지: "예약 확인" 버튼이 연락처 입력 위를 덮음)
- **개발 시**: 스크롤 영역 하단 여백 확보 + 고정 푸터와 콘텐츠 겹침 방지. (달력 R-fix와 동일 패턴)

## R9. "다시 이용하기" 카드 정리
- **요건**:
  1. 최대 **6개**까지만 노출, 나머지는 오래된 순으로 숨김.
  2. 정렬: **즐겨찾기 우선 → 그다음 최신순**.
  3. 카드 **세로 높이를 현재의 약 절반**으로 축소 (너무 큼).

### 구현 메모 (실개발 참고)
- **R1**: `vaccinesSummary(v, species)` — 고양이는 core만 판정. 종 선택 `setPetSpecies()`가 고양이 시 rabies 정리. 펫 시트/상세 카드에서 고양이는 광견병 행 미노출.
- **R2/R3**: `S._lastProvider`에 마지막 수단 저장(실개발=localStorage). `openAuth()`가 이 값으로 로그인/가입 맥락 분기 + 최근 수단을 크게 + 나머지는 "다른 방법으로" 접기. `doLogin()`이 welcome/welcomeBack 토스트 분기. `logout()`은 수단 기억 보존, `resetAll()`은 완전 초기화.
- **R4**: `stepPickup` 진입 시 pickup=null이면 'yes' + 기본 주소 자동 선택.
- **R5**: vDone에 `changeBooking(b.id)` 버튼 추가.
- **R6**: `openServicePicker()`가 최근 예약 1건을 상단 카드로(누르면 마이펫) + 그 아래 신규 서비스 목록.
- **R7**: `refreshFootBtn()`이 disabled + soft 클래스(색) 동시 토글.
- **R8**: `stepPickup` 하단 스페이서 108→170px.
- **R9**: `buildReuseList()` 즐겨찾기 우선→최신순, slice(0,6). `.reuse` CSS 높이 약 절반 축소.
