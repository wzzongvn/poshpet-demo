## 5. 본 개발에서 반드시 지킬 7가지 원칙 (검수에서 도출)

> 상세는 **REQUIREMENTS_UPDATE.md** Part 2 + **SESSION_2026-05-29_FIXES.md** 참조.
> 이 원칙들은 데모에서 실제로 터진 버그의 교훈이다.

1. **모바일 웹 루트에 `text-size-adjust:100%`** — iOS 큰글씨로 레이아웃 깨짐 방지. 그리드는 폰트 독립 고정 높이.
2. **IME(한글·베트남어) 입력 처리** — 버튼 활성/검증에 관여하는 입력은 `oninput`만 의존 금지. composition 이벤트(React: `onCompositionEnd`) 또는 blur/change 보강.
3. **"입력칸 표시 조건"과 "검증 조건"은 단일 진실로 일치** — 어긋나면 입력칸은 보이는데 버튼이 안 켜지는 모순.
4. **단계 전용 로직을 전역 lifecycle(useEffect 등)에 넣을 때 단계 조건 가드 필수**.
5. **긴 입력 폼 단계는 sticky 푸터 대신 흐름 배치/충분한 하단 패딩**.
6. **주소·전화 등 "저장 후 재사용" 항목은 0건→입력→저장→자동선택→추가 흐름 동일 패턴**.
7. **iOS Safari 실기기 QA 필수** — PC/시뮬레이션으로 안 잡히는 버그가 다수.
